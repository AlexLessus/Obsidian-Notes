### Complejidad de un algoritmo / Big O
Dada una variable de entrada, es la función que describe la complejidad de un algoritmo
- Temporal
- Espacial

**Notacion big O**
Es el lenguaje que utilizamos para medir la eficiencia de un algoritmo
Se utiliza para describir la complejidad de un algoritmo, O(n) O(n^2) 
n representa la cantidad de elementos

## Clarifica el problema
**Haz las preguntas que necesites para lograr entender el problema**

- Características de los datos de entrada.
- Características de los datos de salida.

No empezar a codificar hasta entender el problema completamente

Pregunta por los corner cases

>Tip
>Platicarle el problema al entrevistador para ver si entendiste el problema

### Pregunta por la complejidad esperada
Demuestra que te preocupas por la eficiencia tanto en tiempo en memoria de tu solución, es positivo para el entrevistador

### Piensa en voz alta
Practica el estar hablando mientras resuelves el problema, **esta es la unica manera de saber que estas pensando**

**Decir todo el proceso de pensamiento. Te pueden ayudar si te estas equivocando.**

### Codifica una solución 
Si no llegas a la solución perfecta esperada por el entrevistador es mejor programar una respuesta con fuerza bruta y a partir de esta buscar optimizar la solución.
