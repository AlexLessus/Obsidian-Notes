#web 
### Flask - Frontend
Ejecutar python en el navegador
Tiene su propio server para correr python, 
Utiliza métodos GET y POST para interactuar con la pagina

### Ruby on Rails - Backend
Enfocado en backend, te genera todo el código
Es muy robusto, no recomendado para proyectos pequeños
Recomendado para trabajar con bases de datos o proyectos muy grandes
- Active Record: ORM por defecto de rails
- Gems: Sistema de gestion de paquetes
### Express - Backend y Front
Crear servidores web y apis. 
Framework **minimalista** de node.js
- Enrutamiento robusto
- Middleware
- Minimalista
- Extensible
- Rendimiento
- Motor de plantillas 

### Tornado
Corre en python.
Se enfoca en aplicaciones que requieren alta concurrencia y conexiones persistentes en tiempo real.
- Aplicaciones de chat instantánea
- Sistemas de notificaciones en tiempo real
- Juegos multiplicador en tiempo real

AsyncHandler es el componente principal

### Angular
Se utiliza para crear aplicaciones web dinámicas SPA (Single Page Application)
Características:
- Basado en componentes
- Uso de typescript
- Enlace de datos
Funciona conectando lo que el usuario ve con los datos de la aplicación


### Django
Viene con todo integrado. Utiliza POO

- Model     : UI 
- View       : Backend
- Template: Consultas

ORM: Object Relational Mapping
Ya viene integrado sql, tu nunca escibes SQL

Es lentísimo.

Es una sola pieza de python. para agregarle mas cosas es complicado

### Starlette
ASGI: es un estanar para usar asincronismo
- Buen rendimiento
- Python
- Backend puro no proporciona servidor ni DB
- es ligero y flexible


### Meteor
Fullstack basado en JavaScript
**Ofrece un entorno en tiempo real** 
Opera sobre el entorno de Node.js
No utiliza http, utiliza DDP

### Sanic
Genera apis de alto rendimiento, Aplicaciones en tiempo real, Microservicios
Asincrono
Servidor propio


### FastAPI
Para construir apis noshit
Es veloz, Intuitivo y robusto
Documentacion automatica
Valida datos
Seguridad integrada
