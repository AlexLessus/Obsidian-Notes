## Pilares del Diseño Centrado en el Usuario

En el contexto del **TecNM** y los estándares modernos de la industria, el DCU se apoya en tres conceptos fundamentales:

1. **Usabilidad:** Qué tan fácil es para un usuario cumplir su objetivo principal (comprar un producto, registrarse, etc.).
    
2. **Accesibilidad:** Garantizar que la web sea utilizable por personas con diferentes capacidades (cumpliendo estándares como la W3C).
    
3. **Experiencia de Usuario (UX):** El sentimiento o percepción general que tiene una persona al interactuar con tu aplicación.
    

### Las Fases del Proceso

Para que tu **Proyecto Integrador** tenga éxito, debemos seguir estas etapas:

- **Investigación:** Entender quién es el usuario. ¿Es un estudiante, un administrador de bases de datos o un cliente final?
    
- **Concepto:** Definir el flujo de navegación (User Flow) antes de escribir una sola línea de HTML.
    
- **Prototipado:** Crear _wireframes_ para validar la arquitectura de la información.
    
- **Evaluación:** Realizar pruebas de usuario para corregir errores de flujo antes del despliegue en la nube.

## ¿Por qué es vital para un Desarrollador Full-Stack?

Desde la perspectiva de la **Programación del Lado del Cliente (Unidad 3)**, el DCU dicta cómo manipulamos el DOM. No agregamos efectos visuales solo porque "se ven bien", sino para dar _feedback_ al usuario (ej. un spinner de carga mientras una API responde).

> **Regla de Oro:** El usuario no debe pensar. Si una interfaz requiere un manual de instrucciones, el diseño ha fallado.
